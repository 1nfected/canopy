test source

standard test definition (name defined automatically by the test number eg: Test #1) 
test (fun _ ->
    //go somewhere
    //interact with page
    //assert
)

&&& (named test, aliased as ntest) source

standard test definition with a name 
"go somewhere, do somet stuff, assert" &&& fun _ ->
    //go somewhere
    //interact with page
    //assert

//Or
ntest "go somewhere, do somet stuff, assert" (fun _ ->
    //go somewhere
    //interact with page
    //assert
)

&&&& (work in progress, aliased as wip) source

used for debugging. Test runs slower and highlights the elements that it is intereacting with to help debug. If one test is marked wip, only wip tests are ran. Other tests are skipped 
//this is test is run slow and only with other tests marked as wip
"go somewhere, do somet stuff, assert" &&&& fun _ ->
    //go somewhere
    //interact with page
    //assert

//Or
wip (fun _ ->
    //go somewhere
    //interact with page
    //assert
)

&&! (skip, aliased as xtest) source

do not run a test 
//skipped
"go somewhere, do somet stuff, assert" &&! fun _ ->
    //go somewhere
    //interact with page
    //assert

//Or
xtest (fun _ ->
    //go somewhere
    //interact with page
    //assert
)

many source

run a single test X times. Helps with troubleshooting tests that sometimes fail 
many 20 (fun _ ->
    //go somewhere
    //interact with page
    //assert
)

todo source

mark a test as todo to fill in later. LiveHtmlReporter will mark todo tests in the output 
"go somewhere, do somet stuff, assert" &&& todo
